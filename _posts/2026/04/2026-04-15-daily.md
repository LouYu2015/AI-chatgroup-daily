## 方法论

### 从每日1小时到零摩擦：沉稳的猫头鹰87 分享赛博人生录音整理工作流的一年演进

沉稳的猫头鹰87 详细分享了将日均12小时录音（通过 Plaud 采集）自动整理的工作流演进过程，经历三个阶段：

第一阶段：每日手工花1小时下载、上传腾讯会议转录、用 Gemini 总结（当时只有 Gemini 窗口够大）。
第二阶段：改为每周集中处理，尝试浏览器 MCP 和 AI 浏览器（Comet 效果最好但不稳定），复盘总结交给 Cursor 和 CC。
第三阶段：摩擦系数降到0，全自动完成下载→上传→转录→总结→蒸馏。

关键突破因素：1) 24小时不关机的 Mac Mini 提供基础设施；2) Playwright/Chrome 工具稳定性提升；3) 以 AI 为核心而非工程解的思路转变；4) Meta skill 的设计理念——基于目标而非严控 SOP。

这引发了关于语音转录方案（Plaud 内置 vs 本地 Qwen-ASR vs Whisper）、meta skill 写法、ambient agent 等深度讨论。

> 冷静的狐狸41：这个非常有启发！还有skill的撰写也可以直接让AI enforce meta skill。我一般是说
> 沉稳的猫头鹰87：特别是skill这个事情，我创建skill的时候，是用的你之前发的那个metaskill的指导，但人啊，总是忍不住想优化。我是一张嘴，就突然意识到，我在做负优化

### Agent 写代码倾向传统工程解而非 AI 解：如何用 Meta Skill 引导

坚定的海豚06 提出一个痛点：让 Agent 写代码时，理想方案应该是 Agent + Skill + Cron 三位一体的 harness，但 Agent 总是倾向于写传统的 deterministic function，表面上符合设计文档但底层逻辑完全偏离。

沉稳的猫头鹰87 有同感，认为模型训练时没有把 agent 当成 tool 的习惯。冷静的狐狸41 建议在写代码前先让 AI 读 meta skill，用目标导向的思路来写，比较容易解决这个问题。

> 坚定的海豚06：和我给的设计文档大面上看是写了，底层逻辑完全没按我说的来
> 沉稳的猫头鹰87：我理解模型在训练的时候，没有把agent当成tool的习惯。学习的都是古法
> 冷静的狐狸41：比较容易解决，就还是meta skill, 在他写code的之前，直接跟他说，你去读一下meta skill, 用这个思路来写

### 语音转录方案对比：Plaud、Qwen-ASR、Whisper、腾讯会议

围绕沉稳的猫头鹰87 的工作流，群友对比了多种语音转录方案：

- Plaud 内置转录：$20/月 unlimited，坚定的海豚06 推荐，但沉稳的猫头鹰87 表示之前没这么便宜
- Qwen-ASR 1.7B：冷静的狐狸41 推荐，中文友好，比 Whisper 更少 hallucination，可本地运行
- Whisper：2026年略显过时，在长静音段容易出现重复 hallucination
- 腾讯会议：免费且效果好，适合大量录音转文字
- Granola：机智的海豚85 的公司已采购，会议录音好用
